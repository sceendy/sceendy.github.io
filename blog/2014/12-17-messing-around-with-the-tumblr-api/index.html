Messing around with the Tumblr API

December 17, 2014
angular
api
code

In order to avoid managing a blog myself, I wanted to try out handling my blogging with the Tumblr API. I figured it'd be easier to keep up with and leave out any complexity from integrating a blog onto my actual site. It's easier just to hit up some API endpoints with ngResource since I'm using Angular.

This was actually the first time I set up a site to reach out to some API endpoints so I thought I could write a little about my experience.
Tumblr's API documentation is quite simple to understand and allowed for me to get it all hooked up within an hour. I got an OAuth Key and then used Angulars' ngResource to point to their API and get it to return the JSON.
var url = "http://api.tumblr.com/v2/blog/"+ blogurl + "/posts/text?apikey=" + apikey + '&limit=20';
I placed made the blogurl and apikey variables for better visibility and I am not sure if I will keep the URL of the Tumblr I have had since 2009. At the end of the line, I limited the amount of posts to grab to 20 for now to see how I like it. Of course, on the front-end of the home page, I limitTo 4 posts and on the archive page I let it reach the limit set in the URL.

Once I got that setup, I decide to make a little test in the console to make sure all was well.
